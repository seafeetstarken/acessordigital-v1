# Workspace Customization Rules

## 1. Skill Utilization Reporting
Whenever you complete an update, implementation, or refactoring task, you MUST explain to the user exactly which skills were utilized from the available library (e.g., `firebase`, `clean-code`, `frontend-design`, `auth-implementation-patterns`, etc.).

## 2. Standard of Quality Metrics
For each implementation, you must verify against the following quality benchmarks:
- **Zero Mocks**: Ensure no static placeholders exist in dynamic sections.
- **State Reliability**: Verify F5 refresh persistence (reactive data loading).
- **Aesthetic Premium**: Design must feature cohesive palettes, dark mode harmony, and modern typography.
- **Testing Sandbox**: Payment and connection gateways must include visual triggers for sandbox or webhook simulations.
